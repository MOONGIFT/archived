～～　新米プログラマが３日で社内システムを構築しちゃった！？　～～   
======================================================================   
  
pirka は、オリエンタルアーツにとって3代目のWEBシステム構築フレームワークです。   
2003年に初代「HBJ」を発売以来、多くのシステム構築で採用され、進化してきました。   
このたび、大幅なバージョンアップを行うにあたり、名前を pirka (ピリカ) と改め、オープンソースとして公開いたしました。   
  
pirkaとはアイヌ語で「良い、美しい」という意味の単語です。   
既存の様々なフレームワークから、良い点を取り入れ、改善すべき点は改善し、「より良い」フレームワークとなることを目指して命名しています。<!--more-->  
pirka は、MVCモデルを採用しています。   
現在のバージョンでは、View と Controller をサポートしています。   
(将来は Model 部分のサポート機能拡張予定)   
  
pirka は、３つのポリシーを掲げて、実現しています。   
  
１．デザインと制御を分離しよう。   
HTML と java だけで動作するようにする。   
HTML の中にJSPやTaglibのようなHTML以外のものを書き込まない。   
設定ファイルを使わない。   
  
２．Java初心者でも使えるようにしよう。   
java初心者でも迷わずにプログラムを作れるように支援機能を準備する。   
([koropo]というツールを準備しています。)   
  
３．とにかく簡単にしよう！   
インストール不要。   
WEBシステムで必ず実装するような機能は、フレームワークが受け持つ。   
(セッション管理、入力チェック、エラー画面表示、セキュリティ等)   
  
また、Ajax対応や、JUnitでのテスト支援機能など、様々な機能を実現しています。   
既存のフレームワークと比較しても、簡単で高品質を実現できるフレームワークであると、自信を持っております。   
  
先日の第16回ソフトウェア開発環境展(SODEC)に出展し、多数の方から、ご支持の声をいただくことが出来ました。   
いよいよ世の中でも認められる製品が出来たと感じています。   
一人でも多くの方に、このフレームワークを知っていただき、使用していただきたい、また、より良いフレームワークに育てていただきたいと考えております。
